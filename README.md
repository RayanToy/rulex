## 📖 О проекте
**RuLex** — инновационная образовательная платформа, которая использует искусственный интеллект (Claude AI) для автоматической генерации тестовых заданий на проверку словарного запаса.
Платформа решает ключевую проблему: традиционные методы проверки лексики ограничены в точности и объективности. RuLex предлагает **адаптивное тестирование**, где сложность заданий подстраивается под уровень каждого ученика в реальном времени.
> [!NOTE]
> Проект создан при поддержке **Фонда содействия инновациям** в рамках программы «Студенческий стартап» федерального проекта «Платформа университетского технологического предпринимательства».

## ✨ Возможности
### 🎓 Для учеников
- **Адаптивные тесты** — сложность автоматически подстраивается под уровень знаний
- **Поддержка классов 3–11** и выпускников
- **Подробные результаты** — разбивка по частотности слов (высоко-, средне-, низкочастотные)
- **Персональные рекомендации** по развитию словарного запаса
- **История тестирований** — отслеживание прогресса
### 🤖 AI-генерация вопросов
- Автоматическая генерация тестовых заданий через **Claude AI (Sonnet 4)**
- Морфологический анализ слов через **pymorphy3**
- Интеллектуальный подбор **дистракторов** (неправильных вариантов ответа)
- Проверка слов на пригодность для теста (фильтрация географических названий, этнонимов, узкоспециальных терминов)
- Автоматическая коррекция толкований (удаление однокоренных слов)
### 📊 Для преподавателей
- Генерация вопросов: одиночная, массовая и автоматическая
- Ручное добавление и редактирование вопросов
- База вопросов с фильтрацией
- Частотные списки слов по классам

## 🚀 Быстрый старт
### Вариант 1: Docker (рекомендуется)
```bash
git clone https://github.com/RayanToy/rulex.git
cd rulex
echo "ANTHROPIC_API_KEY=ваш_ключ" > .env
docker-compose up --build
```
Откройте http://localhost:8000
### Вариант 2: Локальная установка
```bash
# Клонирование
git clone https://github.com/RayanToy/rulex.git
cd rulex
# Виртуальное окружение
python -m venv venv
source venv/bin/activate        # Linux/macOS
# venv\Scripts\activate         # Windows
# Зависимости
pip install -r requirements.txt
# Настройка
cp .env.example .env
# Отредактируйте .env — добавьте ANTHROPIC_API_KEY
# Запуск
uvicorn main:app --host 127.0.0.1 --port 8000
```
Откройте http://127.0.0.1:8000

## 🧠 Как это работает
### Генерация вопросов (Pipeline)
```
Слово ──► Базовая проверка ──► Проверка LLM ──► Генерация дистракторов ──► Генерация толкования ──► Вопрос
            (pymorphy3)       (Claude AI)        (Claude AI)                (Claude AI)
```
**Этапы:**
| # | Этап | Описание |
|---|------|----------|
| 1 | **Базовая проверка** | Длина, часть речи (только существительные и глаголы), отсутствие спецсимволов |
| 2 | **Проверка LLM** | Claude AI проверяет: не географическое ли название, не этноним, не узкоспециальный термин |
| 3 | **Дистракторы** | Claude AI генерирует 3 слова-гиперонима/гипонима той же части речи (не синонимы, не однокоренные) |
| 4 | **Толкование** | Claude AI создаёт определение без использования целевого слова и дистракторов |
| 5 | **Коррекция** | Автоматическая проверка и перефразирование при обнаружении однокоренных слов |
### Адаптивное тестирование
Алгоритм динамически изменяет сложность:
- **3 правильных ответа подряд** → сложность повышается (+1)
- **2 неправильных ответа подряд** → сложность понижается (−1)
- Диапазон сложности: **1–10**
Распределение вопросов: **70%** высоко/среднечастотные + **30%** низкочастотные.
### Уровни результатов
| Уровень | Условие |
|---------|---------|
| 🟢 **Высокий** | ≥90% общий результат И ≥70% низкочастотные |
| 🟡 **Средний** | ≥70% общий результат |
| 🔴 **Низкий** | <70% общий результат |

## 📡 API-документация
После запуска доступна автоматическая документация:
- **Swagger UI:** http://localhost:8000/docs
- **ReDoc:** http://localhost:8000/redoc
### Основные эндпоинты
#### Аутентификация
| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/auth/register` | Регистрация пользователя |
| `POST` | `/api/auth/login` | Вход в систему |
| `POST` | `/api/auth/logout` | Выход из системы |
| `GET` | `/api/auth/me` | Текущий пользователь |
#### Тестирование
| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/test/start` | Начать адаптивный тест |
| `POST` | `/api/test/complete` | Завершить тест, получить результаты |
| `GET` | `/api/test/history` | История тестирований пользователя |
#### Вопросы
| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/questions` | Все вопросы (краткий формат) |
| `GET` | `/api/questions/full` | Все вопросы (полный формат) |
| `POST` | `/api/questions` | Создать вопрос вручную |
| `PUT` | `/api/questions/{id}` | Обновить вопрос |
| `DELETE` | `/api/questions/{id}` | Удалить вопрос |
#### Генерация
| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/generate-and-save` | Сгенерировать вопрос по слову |
| `POST` | `/api/auto-generate` | Автогенерация 20 вопросов для класса |
| `GET` | `/api/available-classes` | Доступные классы для генерации |

## ⚙️ Конфигурация
### Переменные окружения (`.env`)
| Переменная | Описание | Обязательная |
|-----------|----------|:------------:|
| `ANTHROPIC_API_KEY` | API-ключ Anthropic (Claude AI) | ✅ |
### Данные для генерации (`data/`)
Для работы генератора нужны CSV-файлы:
| Файл | Формат | Описание |
|------|--------|----------|
| `class_N_freq.csv` | `слово,частота` | Частотный список слов для класса N |
| `class_N_relative.csv` | `слово` | Относительный список новых слов для класса N |
| `sharov.csv` | `слово\tчастота` | Частотный словарь Шарова |
Поддерживаемые классы: **2–11** (слова класса N используются для тестирования учеников класса N+1).
